# Monster Image Production Spec

## 目的
- `ChatGPT` にモンスター画像を依頼する時の共通仕様書
- 後で `図鑑 / 編成 / バトル / 合体` に流用しやすい形にする
- 最初は作業量を増やしすぎず、実装に必要な最小セットから始める

## 制作方針
- 世界観は `ダークファンタジー`
- 見た目は `スマホ向けの見やすいドット絵`
- まずは `実装優先`
- 最初から豪華なフルアニメを目指さず、`戦闘に必要な最小限のアニメーション` から作る

## 画像カテゴリ

### 1. バトル用ドット絵
- 用途
  - 戦闘中に表示する本体
- 必須
  - `32x32`
  - 透過背景
  - 正面寄り、またはやや斜め向き
- 推奨
  - 余白は少なめ
  - シルエットで判別しやすい形

### 2. UI用ポートレート
- 用途
  - 編成画面
  - 図鑑一覧
  - 合体候補表示
- サイズ
  - 元画像は `256x256` 以上推奨
  - 正方形
  - 透過背景
- 備考
  - バトル用 `32x32` を無理に拡大しない
  - UI用は別に少し描き込みが多くてもよい

### 3. 図鑑詳細イラスト
- 用途
  - モンスター図鑑詳細
  - モンスター紹介
- サイズ
  - `512x512` 以上推奨
- 優先度
  - 初期は低め
  - 後回しでもよい

## 初期制作で必要なアニメーション

### 最低限
- `Idle`
  - 常時ループ
  - `2〜4フレーム`
- `Attack`
  - 攻撃時
  - `2〜4フレーム`

### 余裕があれば追加
- `Damage`
  - 被弾時
  - `1〜2フレーム`
- `Death`
  - 戦闘不能時
  - `2〜4フレーム`

## 初期実装のおすすめ構成
- 1体ごとにまず作るもの
  - `32x32 Idle 4フレーム`
  - `32x32 Attack 2〜3フレーム`
  - `UI用ポートレート 1枚`
- 後回しでよいもの
  - `Damage`
  - `Death`
  - 図鑑用大イラスト

## 攻撃演出の考え方
- モンスターごとに全エフェクトを個別制作しない
- 共通エフェクトを使う

### 共通化するもの
- 物理攻撃エフェクト
  - 斬撃
  - 打撃
- 魔法攻撃エフェクト
  - 魔法弾
  - 衝撃波
- 属性エフェクト
  - 木
  - 水
  - 火
  - 光
  - 闇
- 仲間化演出
- 合体演出

### モンスター側で必要な差分
- `近距離`
  - 前に出る動き
- `遠距離`
  - その場で撃つ動き
- `物理`
  - 殴る、切る、突く
- `魔法`
  - 詠唱、放つ

## モンスター分類ルール

### 属性
- `木`
- `水`
- `火`
- `光`
- `闇`

### 相性
- `木 -> 水 -> 火 -> 木`
- `光 <-> 闇`
- 有利属性ダメージは `1.5倍`

### 戦闘タイプ
- `近距離`
- `遠距離`

### 攻撃タイプ
- `物理`
- `魔法`

## ステータス仕様
- `HP`
- `攻撃力`
- `魔法攻撃力`
- `防御力`
- `魔法防御力`
- `攻撃速度`

## +値仕様
- 各モンスターに `+値` を持たせる
- `+値` に応じてステータス上昇
- 画像側で `+値差分` は不要
- UI上の数値と表示で対応する

## レア度表現
- レア度は文字ではなく `UI枠色` で表現
- 画像側でレア度専用の描き分けは必須ではない

### レア度対応
- `1 = アイアン`
- `2 = ブロンズ`
- `3 = シルバー`
- `4 = ゴールド`
- `5 = エメラルド`
- `6 = ダイヤモンド`

## 納品形式

### バトル用ドット絵
- 形式
  - `PNG`
- 背景
  - 透過
- サイズ
  - `32x32`
- アニメーション
  - 可能なら `横並びスプライトシート`

### UI用ポートレート
- 形式
  - `PNG`
- 背景
  - 透過
- サイズ
  - `256x256` 以上

## ファイル命名規則

### バトル用
- `mon_<monsterId>_idle.png`
- `mon_<monsterId>_attack.png`
- `mon_<monsterId>_damage.png`
- `mon_<monsterId>_death.png`

### UI用
- `mon_<monsterId>_portrait.png`

### 図鑑用
- `mon_<monsterId>_illustration.png`

- `Assets/Art/Monsters/Battle`
- `Assets/Art/Monsters/Portraits`
- `Assets/Art/Monsters/Illustrations`
- `Assets/Art/Effects/Combat`
- `Assets/Art/Effects/Fusion`

## ChatGPT依頼用の共通条件
- `透過背景`
- `余白少なめ`
- `真正面すぎず少し斜めでも可`
- `32x32でも判別できるシルエット`
- `細部よりも見やすさ優先`
- `ダークファンタジー`
- `スマホゲーム向け`

## ChatGPT依頼テンプレート

### バトル用ドット絵
`ダークファンタジー風のモンスターの32x32ドット絵を作成してください。透過背景、余白少なめ、スマホゲーム向けで、シルエットで判別しやすい見た目にしてください。{モンスター名}、属性は{属性}、戦闘タイプは{近距離/遠距離}、攻撃タイプは{物理/魔法}。今回は{Idle/Attack}用として使うため、自然な待機姿勢または攻撃動作が分かる構図にしてください。`

### UI用ポートレート
`ダークファンタジー風のモンスターUIポートレートを作成してください。正方形、透過背景、スマホゲームの編成画面や図鑑一覧に使える見た目にしてください。{モンスター名}、属性は{属性}、戦闘タイプは{近距離/遠距離}、攻撃タイプは{物理/魔法}。顔や特徴が分かりやすく、背景は入れず、キャラクター中心でお願いします。`

## 初期実装での優先順位
1. 最初の `10〜20体` の `UI用ポートレート`
2. 同じ `10〜20体` の `32x32 Idle`
3. その中の主力モンスターから `Attack`
4. 共通エフェクト
5. 図鑑用イラスト

## 初期制作チェックリスト
- 名前だけ見なくても見分けがつく
- 32x32で潰れすぎていない
- 透過背景になっている
- 余白が大きすぎない
- 属性やタイプの雰囲気が見た目に出ている
- 近距離と遠距離の違いが分かる
- 物理と魔法の違いが分かる
