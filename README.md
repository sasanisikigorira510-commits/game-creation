# モバイル向けゲームアプリ開発メモ

## 目的
iPhone と Android の両方で動く、`魔女狩りの塔`のような雰囲気を参考にしたゲームアプリを開発する。

この文書では、まず作り切りやすい形に落とした `MVP` を前提にまとめる。

## 前提の整理
参考作品の詳細仕様が未確定なので、ひとまず以下を前提にする。

- ダークファンタジー寄りの世界観
- 塔を登る、または階層を攻略する進行
- 短いプレイを繰り返して育成する構成
- スマホ向けのシンプル操作
- キャラ強化、装備、スキルの成長要素

## おすすめの開発方針
個人または少人数で、iPhone/Android 両対応を現実的に進めるなら、第一候補は `Unity`。

理由:

- モバイルゲーム開発の実績が多い
- UI、アニメーション、課金、広告、セーブ実装の情報が多い
- 2D/2.5D の量産がしやすい
- iOS/Android 同時展開がしやすい

別候補:

- `Godot`
  - コストを抑えやすい
  - 軽量で学びやすい
  - ただしモバイル向け運用情報は Unity より少なめ
- `Flutter + Flame`
  - アプリ開発寄りで UI は強い
  - 本格ゲームは Unity より工夫が必要

## 結論
`魔女狩りの塔`系を目指すなら、まずは `Unity 2D` で始めるのが安全。

## MVP のゲーム案
最初の完成目標は、次のような小さめ構成が良い。

### ジャンル
`階層攻略型のセミオートバトル RPG`

### コアループ
1. 塔に挑戦する
2. 敵を倒して階層を進む
3. 素材やゴールドを獲得する
4. キャラや装備を強化する
5. より高い階層へ再挑戦する

### 画面構成
- タイトル
- ホーム
- 編成
- 強化
- バトル
- リザルト
- 図鑑または実績

### MVP 機能
- プレイヤーキャラ 1 体
- 敵 5 から 10 種
- 塔 30 階
- 通常攻撃
- スキル 3 種
- 装備 3 部位
- レベルアップ
- セーブ機能
- 簡易的なガチャまたは報酬箱

## 仕様を作りやすい形に分解

### 1. バトル
- 基本は自動攻撃
- スキルだけ手動発動でもよい
- 1 戦 30 秒から 90 秒程度
- ステータスは `HP / 攻撃 / 防御 / 会心 / 速度`

### 2. 成長
- プレイヤーレベル
- 装備強化
- スキル強化
- 階層到達報酬

### 3. リテンション要素
- デイリー報酬
- 放置報酬
- ミッション
- 実績

最初のバージョンでは、放置報酬とデイリー報酬だけでも十分。

## 開発の順番
おすすめはこの順番。

1. 1 画面で戦闘が成立するか作る
2. 敵を倒して次階層へ進む処理を作る
3. 報酬と強化を作る
4. セーブを入れる
5. UI を整える
6. 演出と音を足す
7. 課金や広告を最後に入れる

## 最初の 2 週間でやる内容

### Week 1
- Unity プロジェクト作成
- バトル画面の土台
- プレイヤー 1 体と敵 1 体
- HP とダメージ処理
- 階層遷移

### Week 2
- 強化画面
- セーブ
- 敵種類追加
- 仮 UI 作成
- Android 実機確認

その後に iPhone 実機確認へ進める。

## 必要なデータ設計
最初は ScriptableObject や JSON で十分。

必要データ:

- キャラクター定義
- 敵定義
- スキル定義
- 装備定義
- 階層定義
- ドロップ定義

## 収益化の入れ方
初期段階では複雑にしすぎない。

- 動画広告で報酬 2 倍
- スタミナ回復
- 初回限定パック
- 広告削除

## 開発上の注意
- 最初からオンライン対戦は入れない
- ガチャ、ギルド、ランキングは後回し
- 演出より先に `遊べるループ` を完成させる
- 片手で遊べるテンポを意識する

## 次のおすすめアクション
このフォルダで次にやるなら、以下のどちらかが良い。

1. `ゲーム仕様書` をさらに詳細化する
2. `Unity プロジェクト用の実装設計` を先に作る

実装に進める場合は、次に以下を作ると良い。

- シーン構成
- クラス設計
- データ定義
- バトルの状態遷移

## 補足
もし参考にしている `魔女狩りの塔` の中で、特に近づけたい要素が次のどれかであれば、設計はかなり変わる。

- バトルテンポ
- 育成の重さ
- 放置要素
- ガチャ要素
- ダークな演出
- ストーリー性

その方向性が固まれば、次に `仕様書` か `Unity 用の設計書` をこのフォルダへ作成できる。
